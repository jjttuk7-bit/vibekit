# 06. Content Operations — VibeKit

## 1. Content Philosophy

VibeKit should not feel like a textbook. It should feel like a box of useful AI kits that users can immediately copy and run.

Every kit should answer:

- What job does this help me do?
- What do I paste into AI?
- What input do I need?
- How do I know it worked?
- When should the AI stop?
- Can I safely reuse this?

## 2. Category System

### Prompt Kits

One-shot or structured prompts for better thinking, writing, planning, and analysis.

Examples:

- Idea to Product Kit
- Expert Council Kit
- Assumption Splitter Kit
- Better Question Kit
- Critique Before Creation Kit

### Context Kits

Reusable context structures that help AI understand projects, users, constraints, and source materials.

Examples:

- Context Pack Builder
- Project Memory Kit
- User Persona Context Kit
- Before Coding Context Kit
- Source Summary Context Kit

### Harness Kits

Execution wrappers that connect AI work to tests, files, commands, logs, and quality gates.

Examples:

- Fix Until Green Kit
- API Debug Harness Kit
- Data Cleaning Harness Kit
- Document QA Harness Kit
- Deployment Check Kit

### Loop Kits

Repeat-until-done structures for improving outputs through review, correction, and verification.

Examples:

- Document Quality Loop
- Rewrite Until Tone Matches
- Research Until Confidence Is High
- Business Idea Validation Loop
- Generate Until Strong Option Appears

### Skill Kits

Templates for turning repeated tasks into reusable agent skills.

Examples:

- Repeated Task to Skill Kit
- Legal Review Skill Kit
- Marketing Analysis Skill Kit
- Data Cleaning Skill Kit
- Personal Second Brain Skill Kit

### Agent Kits

Operating patterns for multi-step agent behavior.

Examples:

- Planner-Executor Agent Kit
- Research-Review-Write Agent Kit
- Tool-Using Agent Kit
- Human Approval Agent Kit
- State-Aware Agent Kit

### Codex Kits

Instructions specifically for coding agents like Codex, Claude Code, and Cursor.

Examples:

- Codex Project Setup Kit
- Codex Fix Until Green Kit
- Codex Refactor Safely Kit
- Codex Add Feature Kit

### RAG Kits

Patterns for retrieval, source checking, citation, and grounded answers.

Examples:

- RAG Source Checker Kit
- Query Rewrite Kit
- Evidence Table Kit
- Hallucination Risk Check Kit
- Citation Quality Kit

### Business Kits

Kits for business idea validation, marketing, sales, monetization, and product strategy.

Examples:

- Idea to MVP Kit
- Customer Pain Finder Kit
- Monetization Check Kit
- Landing Page Copy Kit
- Competitor Benchmark Kit

### Writing Kits

Kits for documents, articles, scripts, reports, and rewriting.

Examples:

- Executive Summary Kit
- Proposal Draft Kit
- YouTube Script Kit
- Blog Outline Kit
- Rewrite for Clarity Kit

## 3. Kit Writing Rules

A kit must:

- Solve one task
- Be copy-ready
- Use original wording
- Include verification
- Include exit condition
- Include clear inputs
- Avoid vague claims
- Avoid unlicensed copying

A kit should not:

- Be a long essay
- Try to solve every problem
- Depend on hidden context
- Use copyrighted source text directly
- Promise guaranteed results

## 4. Standard Kit Format

```text
Title:
Description:
Category:
Best for:
Difficulty:
Use when:
Do not use when:
Required inputs:
Copy-ready block:
Steps:
Verification checklist:
Exit condition:
Tool-specific notes:
Source inspiration:
Reuse status:
```

## 6. Tone Guide

Use:

- Short explanations
- Clear action verbs
- Friendly builder tone
- Copy-ready blocks
- Practical examples

Avoid:

- Academic explanation overload
- “Revolutionary AI” hype
- Too much jargon
- Unsupported claims
- Content that sounds copied from another site

## 7. Kit Quality Score

Score each kit from 1 to 5:

1. Clarity
2. Usefulness
3. Copy-readiness
4. Verification quality
5. Reuse safety

Publish only kits scoring 4 or higher on average.

## 8. Content Update Routine

Weekly routine:

1. Find 5 useful AI sources
2. Check license and reuse status
3. Convert 2 into original kits
4. Improve 3 existing kits
5. Add related kit links
6. Update featured kits
7. Share one kit publicly
