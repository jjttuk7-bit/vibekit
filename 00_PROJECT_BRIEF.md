# 00. VibeKit Project Brief

## 1. Project Name

**VibeKit**

## 2. Short Description

VibeKit is a copy-ready AI kit library for prompts, context packs, harnesses, loops, agents, and reusable skills.

It helps users stop collecting random prompts and start using practical AI execution kits that can be copied, remixed, and run immediately.

## 3. Core Idea

Many people know that AI can help them write, code, analyze, and automate. The problem is that they do not know how to structure the work.

VibeKit solves this by packaging useful AI execution patterns into simple kits.

Each kit contains:

- What the kit is for
- When to use it
- What input is required
- Copy-paste prompt or instruction
- Step-by-step process
- Verification checklist
- Exit condition
- Source inspiration and license note

## 4. Benchmarking Direction

VibeKit is inspired by copy-ready loop and prompt libraries, but it must not copy their wording or content directly.

The benchmarked pattern is:

- Browseable cards
- Clear categories
- Copy-ready blocks
- Short usage explanations
- Practical execution structure
- Optional donation support

VibeKit expands this into a broader concept:

- Prompt Kits
- Context Kits
- Harness Kits
- Loop Kits
- Skill Kits
- Agent Kits
- Codex Kits
- RAG Kits
- Business Kits

## 5. Positioning

VibeKit is not a prompt library.

VibeKit is an AI execution kit library.

Prompt libraries usually provide one-shot text. VibeKit provides reusable operating patterns with context, steps, quality checks, and stop conditions.

## 6. Target Users

Primary users:

- AI builders
- Vibe coders
- Indie hackers
- Prompt engineers
- Codex / Claude Code / Cursor users
- People creating AI skills or agents

Secondary users:

- Marketers
- Content creators
- Product planners
- Researchers
- Non-developers who want copy-ready AI workflows

## 7. Product Personality

VibeKit should feel:

- Friendly
- Practical
- Builder-oriented
- Copy-ready
- Lightweight
- Modern
- Not academic
- Not enterprise-heavy

Avoid:

- Too much jargon
- Overly formal documentation tone
- “Prompt engineering course” feeling
- Fake guru language
- Unsupported claims

## 8. Main Taglines

Primary:

> Everything you need to vibe with AI, copy-ready.

Alternatives:

> Copy-ready AI kits for prompts, context, loops, agents, and skills.

> Pick a kit. Copy it. Run the vibe.

> From random prompts to reusable AI kits.

## 9. MVP Scope

The MVP should include:

- Home page
- Kit browse page
- Kit detail page
- Category filter
- Search
- Source attribution block
- License/reuse badge
- Copy button
- Buy Me a Coffee button
- 30 initial kit cards

The MVP should not include:

- User accounts
- Paid subscription
- Automatic GitHub crawling
- Complex CMS
- AI generation inside the app
- User-submitted content moderation system

## 10. Success Criteria

The MVP is successful if:

- A user can understand the product within 5 seconds
- A user can find a useful kit within 30 seconds
- A user can copy and use a kit without explanation
- Each kit has clear attribution and reuse status
- The site can be expanded by adding new markdown/json kit files

## 11. Long-Term Direction

Future versions can include:

- VibeKit Studio: generate a new kit from a GitHub URL
- VibeKit Packs: downloadable curated packs
- VibeKit for Codex: coding agent instruction packs
- VibeKit Skills: reusable agent skill library
- VibeKit Pro: advanced kits, templates, and exports
