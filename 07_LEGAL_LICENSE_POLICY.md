# 07. Legal and License Policy — VibeKit

## 1. Purpose

VibeKit curates and transforms AI execution patterns from open-source and public materials. This policy exists to prevent unsafe copying, unclear attribution, and misuse of unlicensed content.

This is not legal advice. It is an internal operating policy for safer content creation.

## 2. Core Rule

Do not copy source text directly unless the license clearly allows reuse and attribution is handled correctly.

Prefer:

- Original rewriting
- Pattern extraction
- Summary
- Attribution
- Link to source

Avoid:

- Verbatim copying
- Copying leaked prompts
- Copying proprietary system prompts
- Copying from sources with no license
- Removing attribution from licensed work

## 3. Reuse Status Definitions

### Original

The kit is created from scratch or from general knowledge.

Use this when:

- No specific source was used
- The idea is a general pattern
- The wording is entirely original

### Adapted with Attribution

The kit is inspired by a source that allows adaptation, and attribution is included.

Use this when:

- The source has a permissive license
- The kit was rewritten
- Attribution is visible

### Reference Only

The source can be used for learning or inspiration, but not copied.

Use this when:

- License is unclear
- Source is useful but risky
- Content should be summarized only

### Do Not Copy Directly

The source should not be copied into VibeKit.

Use this when:

- No license exists
- The content may be leaked or proprietary
- The source is a system prompt dump
- The author did not permit reuse

## 4. GitHub License Handling

### MIT

Usually allows reuse, modification, and distribution with attribution and license notice.

Action:

- Attribute source
- Do not imply endorsement
- Rewrite when possible

### Apache-2.0

Usually allows reuse with license notice and other conditions.

Action:

- Attribute source
- Keep license note
- Rewrite when possible

### GPL / AGPL

Can impose stronger obligations when code is reused.

Action:

- Avoid copying code into VibeKit
- Use as reference only unless requirements are clearly understood

### No License

No explicit permission to copy or reuse.

Action:

- Do not copy text or code
- Use only as inspiration
- Write original kit from the general idea

## 5. Attribution Block Template

```text
Source inspiration: [Repository or article title]
Author: [Author or organization]
URL: [Source URL]
License: [License if known]
Reuse note: This VibeKit item is rewritten as an original practical kit and does not copy the source text directly.
```

## 6. Risky Source Types

Avoid using these as direct sources:

- Leaked system prompts
- Proprietary internal prompts
- Scraped paid course content
- Private community templates
- Unlicensed prompt packs
- Brand-specific hidden instructions
- Content that says “do not redistribute”

## 7. User-Facing Disclaimer

Add this to the About or Sources page:

```text
VibeKit curates useful AI execution patterns and rewrites them into original, practical kits. We do not intentionally copy unlicensed source material. If you believe a kit has attribution or license issues, please report it and we will review it.
```

## 8. Internal Publish Checklist

Before publishing a kit:

- Source checked
- License checked
- Reuse status selected
- Text rewritten originally
- Attribution included if needed
- No leaked/proprietary material
- No direct unlicensed copying
- Kit can stand alone without source text

## 9. Takedown Process

If someone reports a license issue:

1. Temporarily mark kit as under review
2. Check source and license
3. Remove copied material if any
4. Add or correct attribution
5. Republish or remove permanently

## 10. Product Principle

VibeKit should win by making source ideas more useful, practical, and safer — not by copying other people’s work.
